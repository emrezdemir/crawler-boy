# 🧠 Memory Bank

This folder is CrawlerBoy's **living memory** — the durable design context that
survives between work sessions. If you (human or AI) are picking the project up
cold, **read these in order**:

1. [`projectbrief.md`](projectbrief.md) — what we're building and why it exists.
2. [`productContext.md`](productContext.md) — the user problem and UX goals.
3. [`systemPatterns.md`](systemPatterns.md) — architecture and key decisions.
4. [`techContext.md`](techContext.md) — stack, constraints, dev setup.
5. [`activeContext.md`](activeContext.md) — what's in focus *right now*.
6. [`progress.md`](progress.md) — what works, what's left, known issues.

## How to maintain it

- After any meaningful change, update **`activeContext.md`** and **`progress.md`**.
- When an architectural decision is made, record it in **`systemPatterns.md`**.
- Keep entries factual and current — stale memory is worse than none.
- The memory bank complements (does not replace) `CHANGELOG.md` and `GRAND_RULES.md`.
